Analysis Environment Setup

This guide walks you through setting up a data analysis environment in Google Colab, using crime data from the City of Chicago. Follow each step carefully—especially running code cells with Shift + Enter—to ensure a smooth experience without errors.

Caution

Each time lines of code are entered in Google Colab Notebooks,
press Shift + Enter to run the code
unless otherwise specified.

The proceeding code will not run otherwise,
and if the walkthrough is followed properly, no errors should occur.

Learn how to set up and run data analysis projects in Google Colab using datasets from the City of Chicago crime database.

This guide will help downloading a dataset, upload it to Google Drive, configure a Colab environment, and access the dataset for persistent use.

Tip

Do you already have Google Colab set up?

Skip down to view the Crime Analysis Reports.

Download the Dataset

Section titled “Download the Dataset”

If we do not have a dataset to work with, this process will ensure we have some data to base the three reports we will write later on.

Go to Chicago Crime Datasets

  1. Select Filter by Date
  2. Select 07/01/2024 to 07/01/2025

  1. Click the dotted hamburger icon attached to a table of data
  2. Click Export Data and download the dataset (file should be ~56MB)

Preserve this crime data CSV file for upload on Setup for Google Colab: 4. Upload the Data below.

Setup for Google Colab

Section titled “Setup for Google Colab”

1. Log in (or create) a Google Account

Section titled “1. Log in (or create) a Google Account”
  • This will become the project account for Google Drive & Google Colab to ensure access and sync to files.

2. Go to Google Colab

Section titled “2. Go to Google Colab”
  1. Click + New notebook
  2. Name the notebook Analysis Environment Setup

Note

A list of reports we will create new Notebooks for:

  • Crime Hotspot Mapping
  • Crime Trend Analysis Over Time
  • Crime Type and Arrest Outcome Analysis

Don’t worry about these reports yet; we will create them when we need to.

3. Configure the Runtime

Section titled “3. Configure the Runtime”
  • Go to Runtime → Change runtime type and confirm Python 3 is selected.

4. Upload the Data

Section titled “4. Upload the Data”

There are two ways to access our files

  1. Upload direct to Google Colab (temporary), or
  2. Upload to Google Drive (persistent)

We will upload to Google Drive to ensure the file is persistently stored; this means the downloaded dataset will be there no matter how many times we reopen Colab.

Using Google Drive, open the Colab Notebooks folder.

  1. Select + New and Upload File
  2. Upload CSV dataset from data.cityofchicago.org

5. Access the Data

Section titled “5. Access the Data”

Before we are able to access the data, we need to mount Google Drive in our Colab Notebook.

  • Move back to Google Colab
  • In a new line on our Notebook, mount Google Drive to access Colab Notebooks files:
  • Once we press Shift + Enter, there will be a few prompts we need to accept to access Google Drive content.

Caution

If we do not Select All on the Google Drive permissions, we will receive an error until all permissions are selected.

6. Load Contents into Dataframe

Section titled “6. Load Contents into Dataframe”

We use the pandas (pd) library to import the data from our .csv dataset into a dataframe (df).
The dataframe df will then be loaded by pd to read the .csv contents using read_csv().

Load file contents into what is called a dataframe.

Before pressing Shift + Enter, move on to the next step to paste the file path.
